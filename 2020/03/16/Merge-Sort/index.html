Merge Two Sorted Arrays Efficiently

Suppose we have two sorted arrays $a[-]$ and $b[-]$, and we want to merge them:

Let’s say a has m entries, and b has n entries. And we should have an array temp which has m+n entries:

The basic idea of merging two arrays is as follows:

  1. In variables i and j we store the current positions in a[-] and b[-] , respectively (starting from i=0 and j=0).

  2. Allocate a temporary array temp[-] for the result.

  3. If a[i] <= b[j] then copy a[i] to temp[i+j] and i++. Otherwise, copy b[j] to temp[i+j] and j++.

  4. Repeat step 2&3 until $i$ or $j$ reaches the end of a[-] or b[-] , respectively, and then copy the rest from the other array.

Note that for the merge sort algorithm, we cannot use iteration unless we use a stack.

As for the time complexity, we have $logn$ levels, and each level we have to merge $n$ elements. Since Each level requires O(n) for merging, the time complexity of merge sort is O(nlogn).

Note that O(nlogn) is the worst/best/average case complexity. The best case is like {1,2,3,4,5,6,7,8}, where the array is already in ascending order. The worst case is like {8,7,6,5,4,3,2,1}, where the array is in decending order. In both cases we need to do the same thing, i.e., splitting and merging, which is in O(nlogn).
